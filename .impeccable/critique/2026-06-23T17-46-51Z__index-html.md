## Priority Issues

- **[P1] Long-page findability still broken.** Sidebar omits `kv-list`, `log-viewer`, `status`, `empty-state`, `dialog`, `toast` (`index.html:750–877`). *Fix:* add every section to nav or add search/filter. *Command:* `/impeccable layout`.
- **[P1] Repeated scaffold dominates.** Same `docs-grid` across ~29 sections (`index.html:53`, `343–1643`). *Fix:* art-directed chapters (Foundations / Core UI / Dense Ops / Feedback / Patterns), each a different composition. *Command:* `/impeccable layout` or `/impeccable shape`.
- **[P1] No adoption accelerators.** No copy buttons, no component search. *Fix:* copy buttons on install/code snippets (`335`, `647–663`); search above nav. *Command:* `/impeccable craft`.
- **[P2] Fake SaaS content dilutes honesty.** Pricing/testimonial/logo cloud (`1455–1575`) read fictional. *Fix:* label as pattern demos or replace with real wrkbnch examples. *Command:* `/impeccable clarify`.
- **[P2] A11y semantics lag visual fixes.** Labels lack `for`/`id` (`505–518`); tabs are anchors without `role="tablist"` (`597–600`); dialog is a `div` not `<dialog>` (`850`); mobile toggle lacks `aria-expanded` (`1746`). *Fix:* real semantic wiring. *Command:* `/impeccable harden`.

## Persona Red Flags

**Developer evaluating adoption:** can't find all components (sidebar gaps); must hand-copy classes (no copy buttons); fake pricing/testimonial raises "is this a CSS package or a fictional SaaS?"

**AI agent / first-time scanner:** hero gives the value prop, but the rest lacks a machine-friendly inventory/search; `mono-label` gives names but not "when to use / class recipe / copy this" structure; long scroll hampers canonical-example extraction.
